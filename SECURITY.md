# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do not open a public issue.**

Instead, email the maintainer or use [GitHub's private vulnerability reporting](https://github.com/bhekanik/openclaw-horror-leaderboard/security/advisories/new).

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 72 hours and work with you on a fix before any public disclosure.

## Scope

This policy covers:
- The web application at openclaw-horror-leaderboard.vercel.app
- The Convex backend functions
- Authentication and authorization logic

Out of scope:
- Third-party services (Convex, Vercel, GitHub)
- Social engineering attacks
- Denial of service attacks

## Supported Versions

Only the latest version deployed to production is supported with security updates.
